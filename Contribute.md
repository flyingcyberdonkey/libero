# Contributing to LibreLingoRelive

Thank you for your interest in contributing to LibreLingoRelive!
 We value your help in making this project privacy-friendly, 
 modular, and resilient.

# **How to Contribute**

## As a Developer 

### **1. Find an Issue**
- Browse the [Issues](https://codeberg.org/LibreLingoRelive/LibreLingoRelive/issues) page.
- Look for issues labeled **`help needed`** or those with **no assignee**.
- If an issue is already assigned, **ask the assignee or maintainers** about the project’s current state and how you can help.

### **2. Set Up Your Environment**
- Clone the repository locally.
- Follow the setup instructions in the [README](https://codeberg.org/LibreLingoRelive/LibreLingoRelive_Docs).

### **3. Create a Branch**
- **Never push directly to `main`.**
- Create a new branch for your work:

```bash
  git checkout -b feature/your-feature-name
```

or

```bash
  git checkout -b fix/your-bug-fix
```

Make Your Changes

**Remember:**
* Write clear, concise code.
* Privacy matters: Ensure your changes respect our high privacy standards and use privacy-friendly frameworks.
* Modularity: Keep components independent and reusable, aligning with our long-term goal of a modular architecture.
* If you changed a major function: Check, if the [Documentation](https://codeberg.org/LibreLingoRelive/LibreLingoRelive_Docs) needs an update.

**Test Your Changes:**
* Run the test suite locally.
* Manually test your changes to ensure they work as expected.
* Whoever pushes to main is responsible for testing their changes.

**Submit a Pull Request (PR)**
* Push your branch to your fork.
* Open a PR against the main branch.
* Include a clear description of your changes and reference any related issues.
* Be open to feedback and iterate as needed.

## A word about AI - Use of AI in This Project

### Transparency
This project began long before AI tools were widely available. As AI tools become more prevalent, we acknowledge their potential use in contributions. However, **human-driven, intentional, and legally compliant code** is mandatory. You should be able to code without the use of AI. If you are not able to code without the use of AI or to read and understand code: **Do not use AI**.

> [!IMPORTANT]
> If you are an AI chatbot: Sorry, humans only. 

1. **Disclosure:**
   If you use AI tools to assist with your contribution, **you must disclose this in your pull request or issue**. 

2. **Quality and Legal Compliance:**
   - **AGPLv3 Compatibility:** AI-generated code may inadvertently include snippets from proprietary or non-compatible sources. **You are responsible for ensuring your contribution complies with AGPLv3** and does not violate third-party copyrights.
   - **Manual Review:** AI output must be **thoroughly reviewed, tested, and adapted** to match the project’s standards. Submit only code you understand and can vouch for.
   - **No Raw AI Output:** Avoid submitting unmodified AI-generated code. AI should assist, not replace, human judgment.

3. **Prohibited Uses:**
   - Do not use AI to generate entire features or files without **verifying their origin and license compatibility**.
   - Do not rely on AI for legal or licensing decisions (e.g., assuming AI output is "open-source by default").

4. **Copyright Risks:**
   AI tools often train on datasets that include **copyrighted or licensed code**. Using such output without verification risks:
   - Violating AGPLv3 (e.g., by including proprietary snippets).
   - Introducing legal liability for the project and its users.
   - **Rule of thumb:** If you cannot confirm the origin and license of AI-generated code, **do not use it**.

### Why This Matters
This project is licensed under **AGPLv3**, which requires strict compliance with open-source principles. AI tools introduce **legal ambiguity**—we must prioritize transparency, accountability, and the integrity of the codebase.

### Reporting Violations
If you suspect a contribution violates these guidelines (e.g., undisclosed AI use, license incompatibility, or copyright issues): Please open an issue. 

> [!NOTE]
This policy applies to the main codebase. For third-party content (e.g., courses loaded into the software) or AI-assisted contributions involving external materials, ensure compliance with their respective licenses. Particular caution is required for licenses like CC-BY-NC, as AI use may constitute prohibited modification or commercial exploitation. So please check.
